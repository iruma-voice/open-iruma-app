# 市議会議員通信簿：評価基準とAIプロンプトについて

本プロジェクト「市議会議員通信簿」は、市民の皆様が市議会の活動をより透明かつ客観的に把握できるよう、公開されている議事録データと選挙公報を基に作成されています。
データの抽出・分類には大規模言語モデル（AI）を活用しており、ここではその評価基準と処理の仕組みを公開します。

## 1. 評価の基本方針（評価ルーブリック）

通信簿は以下の2つの軸で議員の活動を可視化しています。

1. **公約進捗率のトラッキング**
   * **データソース**: 選挙公報に記載された「具体的な政策公約」および本会議の一般質問・答弁の議事録。
   * **判定基準**: 
     * ✅ **進展あり**: 議員の一般質問の中で該当する公約のテーマが明確に取り上げられ、市側から前向きな答弁（実施に向けた検討、現状の課題認識の共有など）が引き出された場合。
     * ⬜ **未着手・確認中**: 該当テーマに関する質問が行われていない、あるいは答弁において明確な進展が見られなかった場合。
   * ※本通信簿は「公約が完全に実現したか」ではなく、「議会の場で公約を推進するための働きかけ（質問・提案）を行ったか」を評価の主軸としています。

2. **注力カテゴリと役割の分類**
   * **データソース**: 各定例会での一般質問の「通告内容」と「実際のやり取り」。
   * **分類方法**: AIが発言内容を読み込み、事前に定義された市の政策カテゴリ（例: `福祉・子育て`, `都市基盤・防災`, `環境・自然` など）および議員の役割（例: `政策提案`, `行政監視`, `課題提起`）を自動的に付与します。

## 2. AI（システムプロンプト）の役割

本システムのバックエンドでは、以下のようなシステムプロンプト（指示書）を用いてAIにデータ処理を行わせています。これにより、人間による恣意的な評価や政治的偏向を排除し、事実ベースの抽出を徹底しています。

> **【システムプロンプトの概要（抜粋）】**
> あなたは公平で客観的なデータアナリストです。提供される市議会の議事録原本を読み込み、以下のタスクを実行してください。
> 1. 議員の質問内容と、それに対する市長・担当部長の答弁の要旨を中立的なトーンで要約すること。
> 2. 発言内容を市の政策カテゴリ一覧（福祉、教育、インフラ等）から最も適切なものに分類すること。
> 3. 対象議員が選挙時に掲げた「公約チェックリスト」と照らし合わせ、今回の質問で触れられている公約があれば「進展あり」としてフラグを立てること。
> 4. いかなる政治的偏向も持たず、事実と議事録に記載された内容のみに基づき判定を行うこと。

## 3. 今後の高度化について
今後は、単純なキーワードマッチングにとどまらず、「他議員が誰も触れていない独自の視点からの提案（ユニークネス）」や「対話の建設性（建設的な議論のスコアリング）」などのより高度な分析を導入していく予定です。
